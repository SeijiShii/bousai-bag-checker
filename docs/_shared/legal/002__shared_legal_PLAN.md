## 2. 実装 Phase 分割 (/flow:tdd 連携)

### Phase 1 (RED→GREEN→IMPROVE): 法務ページ + ルーティング
- 対象: privacy / terms / tokushoho の 3 ページ + /legal/* ルート
- テスト対象: 各ページがレンダリングされ必須項目 (取得情報/免責/運営者/投げ銭任意性) の見出しを含む
- ゴール: 3 ページが表示され必須セクションが存在

### Phase 2: フッタ導線 + 免責表示
- 対象: Footer (法務リンク) + Disclaimer
- テスト対象: フッタから 3 ページへのリンク存在、免責文の表示、投げ銭前確認文 (O43)
- ゴール: 全画面フッタから法務に到達でき、免責が控えめに出る

> 法務本文の実テキストは Markdown/コンテンツで管理し、**公開前に人間が最終確認** (§9.3、論点-S-legal-1)。構造・必須項目・導線は本 Phase で担保。

## 6. リスク・注意点
- **公開前の法務整備が必須** (§9.1): privacy + terms が揃うまでリリースしない (LG-E1)。
- **投げ銭の特商法該当性** (論点-S-legal-1): 表現次第で該当し得る。公開前に人間確認。
- 法務文言は AI ドラフト + 人間レビュー (法的責任は人間)。O38 トーンは維持しつつ法的正確性を優先。
- メイカー文脈フッタは控えめに (§4.8.4、煽らない)。
