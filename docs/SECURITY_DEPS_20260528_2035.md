## 2. Critical / High 詳細 + auto-fix 結果

### スキャン直後の検出 (Critical 5 / High 2)

すべて **`to-ico@1.1.5`** (CF-016 で favicon scaffold 用に追加した 8 年前死蔵 lib) の transitive deps:

| Severity | Package | 経路 | CVE/Advisory 概要 |
|---|---|---|---|
| Critical | `form-data` | to-ico → resize-img → jimp → request → form-data | Unsafe random function for choosing boundary |
| Critical | `jimp@0.2.28` | to-ico → resize-img → jimp | jpeg-js / mkdirp / request via 連鎖、jimp 自体メンテ停止 |
| Critical | `minimist` | to-ico → resize-img → jimp → mkdirp → minimist | Prototype Pollution |
| Critical | `mkdirp` | (上記) | minimist 経由 |
| Critical | `request` | (上記) | Server-Side Request Forgery |
| High | `jpeg-js` | (上記) | Infinite loop / Uncontrolled resource consumption |
| High | `url-regex` | (上記) | ReDoS |

### Root cause + auto-fix

- **Root cause**: `to-ico@1.1.5` (2017-04 最終公開、8 年メンテなし) が古い `jimp` を引きずる
- **対応**: 直接依存 1 個 `to-ico` を `png-to-ico@3.0.1` (2025-08 maintained) に置換
- **コード変更**: `scripts/gen-favicon.ts` の `import toIco from 'to-ico'` → `import pngToIco from 'png-to-ico'` + 関数名 1 箇所
- **検証**:
  - `npm run gen:favicon` 再実行 → favicon.ico multi-res (16/32/48) 正常生成 (`file` で `MS Windows icon resource - 3 icons` 確認)
  - `npm run typecheck` pass
  - `npm test` → 168/168 pass 維持
