## 1. デザイン原則(コンセプト → 視覚の翻訳)

1. **不安を煽らない** — 災害恐怖・緊急性の演出をしない。期限切れも「赤い警告」でなく落ち着いたムード色で淡々と伝える。(concept §1「不安を煽らず淡々と」)
2. **整頓された安心** — 余白広め・情報密度低め。「備えが整っている」状態が一目で分かる。緑=鮮度OK の視覚的な気持ちよさを設計の軸にする。
3. **淡々とした信頼感** — 装飾過多にしない。中立的なサンセリフ + 落ち着いた緑系で、生活に馴染む実用ツールのトーン。
4. **低頻度利用に優しい** — 年数回〜月数回しか開かないため、再訪時に迷わない明快な情報階層と、状態(鮮度)の即時把握を優先。
5. **入口で「これは何？」が分かる** — リンク流入の初見者がファーストビューでサービスの正体を理解できる(O41)。
6. **課金は誠実に** — 有料機能(買い物 TODO リスト)の価格(100円)と対価を、購入 CTA より前に明示する(O43、ダークパターン回避)。

## 2. カラートークン(意味名 + 値 + 用途)

| トークン | 値 | 用途 |
|---|---|---|
| `--color-primary` | `#2E8B74` | 主色(ボタン/リンク/アクティブ)。落ち着いた緑ティール |
| `--color-primary-dark` | `#1F6B58` | hover/active、強調 |
| `--color-primary-fg` | `#FFFFFF` | primary 上の文字 |
| `--color-bg` | `#F7F8F6` | アプリ背景(緑みオフホワイト) |
| `--color-surface` | `#FFFFFF` | カード/パネル面 |
| `--color-surface-muted` | `#EFF2EE` | 副次面/区切り背景 |
| `--color-text` | `#2A2F2C` | 本文(暗い緑グレー) |
| `--color-text-muted` | `#6B746F` | 補助テキスト/ラベル |
| `--color-border` | `#E2E6E2` | 境界線/区切り |
| `--color-fresh` | `#4CA085` | 状態: 鮮度OK(期限に余裕) |
| `--color-warn` | `#C98A3B` | 状態: 期限が近い(落ち着いた琥珀、純黄でなく煽らない) |
| `--color-warn-bg` | `#FBF3E6` | warn の淡い背景 |
| `--color-expired` | `#C25B4E` | 状態: 期限切れ/要交換(テラコッタ、**純赤は使わない**) |
| `--color-expired-bg` | `#FBEDEA` | expired の淡い背景 |
| `--color-focus` | `#2E8B74` | フォーカスリング(primary と同系) |

**原則**: 画面は生 hex を直書きせず必ずトークンを使う。状態色(fresh/warn/expired)は「鮮度の3段階」の視覚言語として統一。**純赤(#F00 系)・点滅・赤バッジ多用は禁止**(不安喚起、原則1)。

**コントラスト**: text/bg、primary-fg/primary は WCAG AA(4.5:1)を満たすこと。warn/expired をテキスト色に使う場合は背景とのコントラストを確認(淡背景 + 濃文字の組合せを推奨)。

## 3. タイポグラフィ

| 役割 | 指定 |
|---|---|
| フォント(JP/EN) | 中立サンセリフ。`Inter` + `Noto Sans JP` を Web フォント、フォールバックに system-ui / sans-serif |
| `--font-size-xs` | 12px(キャプション/メタ) |
| `--font-size-sm` | 14px(補助/ラベル) |
| `--font-size-base` | 16px(本文) |
| `--font-size-lg` | 20px(小見出し) |
| `--font-size-xl` | 24px(画面タイトル) |
| `--font-size-2xl` | 32px(ヒーロー/入口見出し) |
| 行間 | 本文 1.6、見出し 1.3 |
| 字面 | 詰めすぎない。低頻度利用 + シニア層も想定し base 16px を下限 |

中立サンセリフ採用の根拠: 読みやすさ最優先(concept §4.9)、生活ツールの淡々としたトーンに装飾書体は不適。

## 4. 形・影・余白

| トークン | 値 | 用途 |
|---|---|---|
| `--radius-sm` | 6px | チップ/小要素 |
| `--radius-md` | 10px | ボタン/入力 |
| `--radius-lg` | 16px | カード/モーダル |
| `--shadow-sm` | `0 1px 2px rgba(42,47,44,.06)` | 軽い浮き(カード) |
| `--shadow-md` | `0 4px 12px rgba(42,47,44,.10)` | モーダル/ポップオーバー |
| `--space-1`..`--space-8` | 4/8/12/16/24/32/48/64px | 余白スケール(8px グリッド基調) |

**原則**: 余白は広めに取り「整頓された安心」(原則2)を視覚化。角丸は柔らかめ(穏やかさ)。影は控えめ(淡々)。

## 5. コンポーネント(基本形)

- **ボタン**: primary(塗り)/ secondary(outline)/ ghost(文字のみ)。radius-md、最小タップ領域 44×44px(モバイル)。
- **カード**: surface + shadow-sm + radius-lg + space-5 padding。品目カード/点検カードの基本器。
- **状態チップ**: fresh/warn/expired を淡背景 + 濃文字 + 小アイコンで。鮮度の3段階を一目で。
- **入力**: ラベル常時表示、border + focus ring(color-focus)、エラーは expired 色 + 補足文(点滅させない)。
- **ナビ**: モバイルはボトムタブ(品目/点検/買い物/設定)、デスクトップは左サイド or トップ。
- **空状態**: 自作 SVG イラスト(line-art、テーマ色追従)+ 一言 + 次アクション。(例: 品目0件「まだ何も登録されていません。最初の品目を追加しましょう」)
- **入口の「これは何？」導線(O41)**: 入口/ヒーローに**簡潔なリード文**を最優先で配置(例:「防災備蓄の賞味期限・電池切れを、季節ごとに淡々と点検できるアプリです」)。デザイン上リード文が収まらない画面では、**常設インフォボタン(丸付き「?」)+ 軽量モーダル**を標準化(内容は concept §1 から導出、O38 準拠)。
- **進捗体験(O45)**: 本サービスは本質遅延(>1〜2s)のある重い AI/アップロード処理が中心ではない(品目 CRUD は軽量)。写真アップロード(R2)や cron 通知の手動トリガー等で待ちが出る箇所のみ、素のスピナーでなく段階文言(「写真を保存しています…」)を出す。嘘進捗・わざと遅延は禁止。

## 6. ボイス & コピー(perspectives O38 準拠)

**トーン**: 淡々・穏やか・急かさない。「〜しましょう」程度の柔らかい促し。煽り(「今すぐ!」「危険です!」)は禁止(charter §2.2、concept §4.8.2)。

**技術用語 NG → 言い換え**(ユーザー向け文字列のみ。コメント/型/ログ/テストは対象外):

| NG(技術用語) | 言い換え |
|---|---|
| マイグレーション / DB | (UI に出さない) |
| 認証 / ログイン状態 | 「アカウント」「サインイン」 |
| バリデーションエラー | 「入力を確認してください」 |
| キャッシュ / 同期失敗 | 「うまく保存できませんでした。もう一度お試しください」 |
| アンロック / ペイウォール | 「買い物 TODO リストを使う(100円)」 |
| トークン / API / エンドポイント | (UI に出さない) |

**エラーメッセージ方針**: 何が起きたか + どうすればよいかをセットで。DB 内容・スタックトレース・内部 ID を出さない(SEC-002 とも整合)。淡々と、ユーザーを責めない。

**期限表現**: 「期限切れです」より「交換の時期です」「点検どうぞ」のような淡々とした促し(原則1)。

> 仕上げの対話校正は `/flow:wording`(O42)で JA 確定 → EN 訳整合。本 SoT はトーン基準を提供する。

## 7. アイコン & イラスト戦略

- **UI アイコン**: OSS アイコンセット(例: lucide)。**絵文字は使わない**(環境依存で崩れる + 一貫性欠如)。
- **ブランド表現/空状態/入口ヒーロー**: 自作 SVG イラスト(line-art、`currentColor`/テーマ色追従、no-key)。防災備蓄の「整頓された棚」「チェック済みリスト」等を穏やかに。
- **写実ラスター画像**: 原則使わない(必要時のみ任意・要キーのエスケープハッチ)。品目写真はユーザーアップロード(R2)で別レイヤー。
- **アイコンの意味**: 鮮度3段階(fresh=チェック/葉、warn=時計、expired=リフレッシュ/交換)を状態色と組で。

## 8. レビュー基準(視覚 + コピー、Step 4 で使用)

**視覚(O34 Level 3、headless スクショをマルチモーダル批評)**:
- [ ] 情報階層が明快(画面の主役が一目で分かる)
- [ ] 余白が広めで「整頓された安心」が出ている
- [ ] 状態色(fresh/warn/expired)がトークン通り、純赤・点滅・煽り表現がない
- [ ] コントラスト WCAG AA を満たす
- [ ] トークン経由(生 hex 直書きがない)
- [ ] 絵文字を UI アイコンに使っていない

**コピー(O38)**:
- [ ] ユーザー向け文字列に技術用語が混入していない(deny-list grep)
- [ ] トーンが淡々・穏やか(煽りがない)
- [ ] エラーが「何が + どうすれば」になっている

**入口理解(O41)**:
- [ ] 入口ファーストビューで初見者がサービスの正体を言える(リード文 or インフォボタン)

**価格透明性(O43、課金 PJ)**:
- [ ] 買い物 TODO リストの購入導線で「100円 + 何ができるか」が CTA より前・ファーストビュー内に明示
- [ ] 価格が Checkout 最後で初出しになっていない(ダークパターン回避)
